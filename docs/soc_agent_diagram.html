What This Agent Does

üéØ Purpose

This is an autonomous AI agent that analyzes legal case documents to generate a structured Statement of Claim (SOC) issues table. Unlike a fixed workflow, this agent makes intelligent decisions about which tools to use and in what order based on the user's request.

The agent is designed to:

  • Read and understand case documents
  • Identify the claimant and defendant statements
  • Generate a structured issues table
  • Save results for further processing

ü§ñ How It Works

The agent follows a ReAct (Reason + Act) pattern:

  • Receives a user request
  • Thinks about what action to take
  • Selects an appropriate tool
  • Observes the result
  • Repeats until the task is complete

The agent can loop through these steps multiple times, autonomously deciding when to use each tool.

Example: Typical Agent Execution Flow

  1. Step 1: Receive Request

    "Generate statement of claim issues from the case files"

  2. Step 2: List Documents

    Agent calls list_documents() to see available files

  3. Step 3: Get Document Details

    Agent calls get_all_document_details() to understand each file

  4. Step 4: Read Key Documents

    Agent calls retrieve_document("O - Statement of Claim.md") and retrieve_document("P - Statement of Defence.md")

  5. Step 5: Generate Issues

    Agent calls generate_soc_issue_table() with the statements and document details

  6. Step 6: Save Output

    Agent calls write_court_issues_json() to save the structured issues
